What is Containerization?

Containerization is the process of packaging an application and its dependent resources into a container. The container produces an image, which is an executable package of software that includes everything that is needed to run the application.

A container engine, such as Docker, is layered on top of the host OS, as you can see in the image below. Containers do not have guest OSs; instead, the container engine runs the application. A container uses minimal resources, as it only includes what is necessary to the execution of the application, and because a container does not rely on a specific OS, it can be easily transferred to a different host without OS compatibility issues.

Containers are separate from one another, so there is little or no access between them. The applications stored in containers are specific to the library of resources included in that container. Containerization creates very little wastage of resources, and deployments are faster and more reliable.
