What is Virtualization?

Virtualization allows one host to run multiple operating systems (OSs) in different virtual machines (VMs). In the image below, you can see that there is a hypervisor running on top of the host OS. The hypervisor pulls resources from the physical hardware and allocates them to the virtual environment. In this example, there are three guest OSs on top of the hypervisor, and these are the VMs.

A VM can be defined as a software-based computer. Like a physical computer, a VM has an OS and applications. VMs are independent of one another, so you can run different OSs (such as Windows, Mac OS, Linux and Unix) in separate VMs but all on the same host. Since VMs are independent, you can also move them to different hosts. This creates flexibility and portability, which is useful in the case of failure in the hosting infrastructure. Additionally, since you can run multiple VMs from one physical infrastructure, costs are reduced on system components such as hardware, maintenance and electricity.

Some disadvantages to virtualization are that the running of multiple VMs can cause unstable performance, hypervisors are not always as efficient as host OSs, and there is a long boot-up time.
